# 🚀 Zap Pilotの仕組み

## 🧠 コアメカニズム

Zap Pilotは、**感情主導型のポートフォリオ・オートパイロット**です。**恐怖・強欲指数（Fear & Greed
Index）**を使用して、ポートフォリオのどれだけを**暗号資産（BTC/ETH）**にし、どれだけを**ステーブルコイン**にするかを自動的に決定します。

### 主要な原則

- **恐怖で買う**：市場が怯えているとき、暗号資産の配分を増やします。
- **強欲で守る**：市場が熱狂しているとき、利益をステーブルコインに確定させます。
- **ノンカストディアル**：すべての動きにあなたが署名します。私たちは決してあなたの資金を保有しません。

---

## 1️⃣ 接続と入金

ウォレット（MetaMaskなど）を接続し、資金を入金します。ランダムな戦略を選ぶ代わりに、あなたは私たちの**レジーム・ベース・モデル（Regime-Based
Model）**に参加することになります。

---

## 2️⃣ レジーム（市場区分）

あなたの資金は、5つの市場レジームに基づいて配分されます：

| レジーム (Regime) | 指標 (Indicator) | 戦略 (Strategy)                   |
| :---------------- | :--------------- | :-------------------------------- |
| **極端な恐怖**    | 🔴 FGI < 25      | **70% 暗号資産** / 30% ステーブル |
| **恐怖**          | 🟠 FGI 25-45     | **60% 暗号資産** / 40% ステーブル |
| **中立**          | 🟡 FGI 46-54     | **50% 暗号資産** / 50% ステーブル |
| **強欲**          | 🟢 FGI 55-75     | **40% 暗号資産** / 60% ステーブル |
| **極端な強欲**    | 🟩 FGI > 75      | **30% 暗号資産** / 70% ステーブル |

**詳細はこちら：** [詳細な戦略ガイド](../strategies)

---

## 3️⃣ 📐 ケリー配分 (Kelly Allocation)

私たちが使用する*具体的*なプロトコル（例：ステーブルコインにはAaveかCompoundか、LPにはUniswapかCurveか）を最適化するために、
**ケリー基準（Kelly
Criterion）**を使用します。これにより、スマートコントラクトのリスクを最小限に抑えながら、利回りを最大化します。

**詳細な説明：** [ケリー基準の詳細](../strategies/kelly-criterion)

---

## 4️⃣ ♻️ リバランシング（ユーザー主導）

市場のレジームが移行する（例：恐怖 → 中立）と、**リバランス提案**が生成されます。

1.  **通知**：アラート（メール/カレンダー）を受け取ります。
2.  **実行**：Zap Pilotのダッシュボードで「リバランス」をクリックします。
3.  **段階的な実行**：ロジックは取引を5〜10日間に分割し、スリッページとタイミングのリスクを最小限に抑えます。

---

## 🔒 セキュリティと透明性

- **ノンカストディアル**：資金は常にあなたのウォレットに残ります。
- **オンチェーン実行**：すべての取引は透明です。
- **ユーザー管理**：すべての動きをあなたが承認します。

**セキュリティ詳細：** [セキュリティインシデントと透明性](../security)
